# Tech Article Polisher - 质量控制标准

## 技术准确性验证

### SQL 脚本验证

**验证清单：**
- [ ] SQL 语法正确性
- [ ] 视图名称准确性（V$、DBA_、GV$ 等）
- [ ] 字段名称准确性
- [ ] 函数名称准确性
- [ ] 版本兼容性标注

**版本兼容性标注：**
```sql
-- 适用版本：Oracle 11g / 12c / 19c / 26ai
-- 适用版本：Oracle 12c / 19c / 26ai（多租户特性）
-- 适用版本：Oracle 23ai / 26ai（向量特性）
```

### 技术概念验证

**验证清单：**
- [ ] 技术术语准确性
- [ ] 概念解释正确性
- [ ] 版本差异说明
- [ ] 参数名称准确性

**常见错误示例：**
- ❌ 错误：V$SESSION_LONGOPS 在 11g 中不存在
- ✅ 正确：V$SESSION_LONGOPS 在 11g/12c/19c/26ai 中都存在
- ❌ 错误：CON_ID 在 11g 中可用
- ✅ 正确：CON_ID 在 12c 及以上版本可用（多租户特性）

## 文学手法使用规范

### 使用频率控制

**轻度润色：**
- 使用 2-3 种文学手法
- 每篇文章 3-5 处应用
- 保持原文风格

**标准润色：**
- 使用 5-7 种文学手法
- 每篇文章 8-12 处应用
- 适度增强可读性

**深度润色：**
- 使用 8-10 种文学手法
- 每篇文章 15-20 处应用
- 显著提升可读性

### 手法选择原则

**优先级排序：**
1. **必须使用**：物化手法、比喻与类比、场景化叙事
2. **推荐使用**：数字化表达、对比与反差、设问与自答
3. **可选使用**：金庸式笔触、借古喻今、列锦手法
4. **谨慎使用**：悬念与伏笔、跳笔、逆喻

**避免过度使用：**
- 同一手法在同一篇文章中不超过 5 次
- 避免连续使用多种手法
- 确保手法自然融入，不生硬

## 标题生成标准

### 标题质量评估

**优秀标题特征：**
- 包含数字（如"3 个技巧"、"5 步排查"）
- 使用疑问句（如"为什么建了索引却不用？"）
- 使用对比（如"从 11g 到 26ai"）
- 包含关键词（如"性能调优"、"索引优化"）
- 长度适中（15-25 字）

**标题生成模板：**

**模板 1：数字 + 关键词**
```
{数字}个{关键词}，{结果}
示例：3 个指标，一眼看穿索引健康度
```

**模板 2：疑问句 + 关键词**
```
为什么{问题}？{答案}
示例：为什么建了索引却不用？聚簇因子告诉你
```

**模板 3：对比 + 关键词**
```
从{旧版本}到{新版本}：{主题}
示例：从 11g 到 26ai：DBA 的十年进化
```

**模板 4：场景 + 关键词**
```
{场景}时，{解决方案}
示例：生产系统卡死时，这 3 个视图救你命
```

**模板 5：总结 + 关键词**
```
{主题}：{核心价值}
示例：索引体检报告：DBA_INDEXES 的秘密
```

## 输出质量检查

### 润色前后对比

**检查项：**
- [ ] 技术内容准确性保持
- [ ] 文章结构完整性保持
- [ ] 可读性显著提升

- [ ] 格式规范统一

### 常见问题修复

**问题 1：过度润色**
- 症状：文学手法过多，影响技术内容
- 修复：减少文学手法使用频率，保持技术内容清晰

**问题 2：技术错误**
- 症状：SQL 脚本语法错误，视图名称错误
- 修复：验证技术准确性，修正错误内容

**问题 3：格式混乱**
- 症状：标题层级混乱，格式不统一
- 修复：统一格式规范，调整标题层级

**问题 4：标题质量低**
- 症状：标题不够吸引人，缺少关键词
- 修复：使用标题生成模板，优化标题质量
